# BTC Intraday Daily Opportunity Signal

本策略用于 BTCUSDT 永续合约，目标是把较稳定的高周期趋势主仓和更频繁的日内小仓机会结合起来。它不是单纯为了交易次数而频繁追涨杀跌，而是在 4h 趋势主模型之外，增加一个 1h 日内补位模型，让策略在主趋势仓位较小或空仓时仍然有较多开仓机会。

## 策略逻辑

趋势主模型负责主要收益来源。它用较慢的价格结构、动量延续、均线相对位置、反弹强弱和波动率来判断 BTC 是否处于更清晰的趋势状态。当下行结构更强、反弹受阻且波动率可控时，策略倾向于建立空头趋势仓位；当恢复结构足够干净时，策略只允许较小的多头恢复仓位。

日内补位模型只在趋势主仓较小或接近空仓时启用。它观察价格相对短期成交价值的位置、日内波动、短周期 RSI 状态和成交流方向。当价格短线偏离较大且出现均值回归条件时，模型用很小的仓位尝试做多或做空。这个补位仓位有持仓时间限制，目的是提高开仓覆盖率，而不是取代趋势主仓。

## 开仓

趋势仓位在 4h 结构确认后开仓。空头需要弱动量、价格低于关键趋势参考、反弹没有破坏结构，并且波动率不超过上限。多头只在恢复行情质量较好时以较小仓位参与。

日内仓位在 1h 条件满足后开仓，并通过 15m bar 执行。只有当主趋势仓位很小或没有主趋势仓位时，日内补位才会参与，避免和主趋势仓位反复打架。

## 平仓

策略通过目标仓位调整平仓。趋势结构失效、反弹破坏空头结构、恢复行情变弱或波动率不适合继续持仓时，目标仓位会下降到零附近。日内补位仓位有最大持仓时间，机会过期后会主动退出。

## 参数

`leverage` 控制杠杆倍数，越高收益和回撤都会被放大。`margin_budget` 是策略使用的保证金预算，也是平台展示收益率的重要分母。`weight_scale` 控制趋势主仓的强度。`max_effective_exposure` 限制杠杆后的最大名义敞口。`overlay_cap` 控制日内补位仓位大小，越大交易机会更明显，但手续费和震荡亏损也更高。`overlay_z_entry` 控制日内补位触发难度，越低越容易开仓，越高越保守。

## 风险

主要风险包括 BTC 急涨急跌、重大消息跳空、震荡市场中日内补位连续被打损、实际手续费或滑点高于回测假设、资金费率偏离、平台成交模型与真实成交不同。更高交易频率会天然增加成本压力，因此这个版本虽然提高了每日开仓机会，但仍需要关注净收益和最大回撤，而不是只看交易次数。
